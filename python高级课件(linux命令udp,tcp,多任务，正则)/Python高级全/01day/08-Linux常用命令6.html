Linux 终端命令(gedit,more,管道,cat)

1. 文本编辑器：gedit

这是 Linux 下的一个纯文本编辑器,但你也可以把它用来当成是一个集成开发环境 (IDE), 它会根据不同的语言高亮显现关键字和标识符

parallels@parallels-vm:~/Desktop$ gedit hm.py

2. 分屏显示：more

查看文件内容时，在信息过长无法在一屏上显示时，会出现快速滚屏，使得用户无法看清文件的内容，此时可以使用more命令，每次只显示一页

parallels@parallels-vm:~/Desktop$ more hm.py
快捷键 作用
空格键 显示下一页
q键 退出
h键 获取帮助

3. 管道：|

管道：一个命令的输出可以通过管道作为另一个命令的输入。

管道我们可以理解现实生活中的管子，管子的一头塞东西进去，另一头取出来，这里“ | ”的左右分为两端，左端塞东西(写)，右端取东西(读)。

4. 建立链接文件：ln

Linux链接文件类似于Windows下的快捷方式。

链接文件分为软链接和硬链接:

  • 软链接：软链接不占用磁盘空间，源文件删除则软链接失效。
  • 硬链接：硬链接只能链接普通文件，不能链接目录。

使用格式：

# 建立硬链接
ln 源文件 链接文件
# 建立软链接
ln -s 源文件 链接文件

5. 查看或者合并文件内容：cat

5.1 查看文件内容 

# 查看当前目录下的文件内容
parallels@parallels-vm:~/Desktop/itheima$ cat hm.txt 

# cat hm.txt文件中的内容
   《劝学》 
       —颜真卿
三更灯火五更鸡,
正是男儿发愤时。 
黑发不知勤学早,
白首方悔读书迟。

# 查看绝对路径下的文件内容
parallels@parallels-vm:/$ cat /etc/host.conf

# host.conf文件中的内容
# The "order" line is only used by old versions of the C library.
order hosts,bind
multi on

5.2 合并文件内容 

# 把hm1.txt和hm2.txt文件的内容都重定向到hm.txt中
parallels@parallels-vm:~/Desktop/itheima$ cat hm1.txt hm2.txt > hm.txt

# 查看文件内容
parallels@parallels-vm:~/Desktop/itheima$ cat hm.txt 
11-22-33-44-55-66-77-88-99
99-88-77-66-55-44-33-22-11
